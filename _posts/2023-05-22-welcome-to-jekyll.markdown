정적인 블로그는 복잡한 기능 없이도 빠르게 글을 남길 수 있다는 점이 좋습니다.
Jekyll은 구조가 단순해서 오래 두고 관리하기에 부담이 적고, GitHub Pages와도 잘 맞습니다.

무엇보다 글을 쓰는 흐름이 단순합니다.
마크다운 파일 하나를 추가하고 커밋하면 배포까지 자연스럽게 이어지기 때문에, 기록 자체에 집중하기 쉽습니다.

처음에는 테마 기본 화면으로 시작했지만, 결국 중요한 건 꾸준히 남기는 내용이라고 생각합니다.
앞으로는 다음 기준으로 블로그를 채워 보려고 합니다.

- 짧아도 실제로 도움이 되는 기록일 것
- 문제와 해결 과정을 함께 남길 것
- 시간이 지나 다시 봐도 이해할 수 있게 쓸 것

아주 거창한 목표보다, 하나씩 꾸준히 쌓는 쪽이 더 현실적입니다.
